# 电商价格监控爬虫 (CR001) 使用说明

## 概述

电商价格监控爬虫是一个专门用于监控各大电商平台商品价格的爬虫工具，支持淘宝、京东、拼多多和亚马逊（中国）等主流电商平台。

## 功能特点

1. **多平台支持**: 淘宝、京东、拼多多、亚马逊（中国）
2. **动态页面处理**: 使用Selenium处理JavaScript渲染
3. **智能解析**: 自动提取商品标题、价格、销量等信息
4. **价格过滤**: 支持按价格范围筛选商品
5. **数据导出**: 自动生成格式化的Excel文件

## 系统要求

### 硬件要求
- **内存**: 至少4GB RAM

## 快速使用

### 方法1: 命令行演示
```bash
# 运行电商爬虫演示
python main.py --demo-ecommerce

# 实际运行电商爬虫
python main.py --run ecommerce -- --platforms taobao jd --category electronics --max-pages 3
```

### 方法2: Python代码
```python
from crawlers.ecommerce.ecommerce_crawler import EcommerceCrawler

# 创建爬虫实例
crawler = EcommerceCrawler()

# 运行爬虫
data = crawler.run(
    platforms=["taobao", "jd"],  # 平台列表
    category="electronics",       # 商品分类
    max_pages=3,                  # 最大页数
    products_per_page=20,         # 每页商品数
    price_range={"min": 1000, "max": 10000}  # 价格范围
)

print(f"爬取到 {len(data)} 个商品")

# 查看示例数据
for i, product in enumerate(data[:5], 1):
    print(f"{i}. {product['title']} - ¥{product['price']:.2f}")
```

## 输出数据格式

### Excel文件结构
```
商品数据_YYYYMMDD_HHMMSS.xlsx
├── 爬取数据工作表
│   ├── product_id: 商品唯一ID
│   ├── platform: 平台名称
│   ├── title: 商品标题
│   ├── price: 当前价格
│   ├── original_price: 原价（如有）
│   ├── sales: 销量
│   ├── shop_name: 店铺名称
│   ├── url: 商品链接
│   └── crawl_time: 爬取时间
├── 爬虫统计工作表
│   ├── 总数据记录数
│   ├── 请求成功率
│   ├── 运行时间
│   └── 详细统计信息

├── 数据概览工作表
│   ├── 数据样本
│   ├── 字段统计
│   └── 数据摘要
```

### 数据字段说明
| 字段 | 类型 | 说明 |
|------|------|------|
| product_id | string | 商品唯一标识符（MD5哈希） |
| platform | string | 电商平台名称 |
| title | string | 商品标题（截断至100字符） |
| price | float | 当前价格（人民币） |
| original_price | float | 原价（如有促销） |
| sales | int | 销量（已处理单位换算） |
| shop_name | string | 店铺名称 |
| url | string | 商品链接 |
| crawl_time | string | 爬取时间戳（YYYY-MM-DD HH:MM:SS） |

## 注意事项

### 反爬虫策略
1. **请求频率**: 避免过快请求，建议设置延迟
2. **User-Agent**: 使用随机User-Agent
3. **代理**: 建议使用代理IP池
4. **Cookies**: 模拟真实用户行为

### 法律合规
1. **robots.txt**: 遵守目标网站的robots.txt规则
2. **数据使用**: 仅用于个人学习研究
3. **隐私保护**: 不获取用户隐私信息
4. **版权尊重**: 不侵犯平台知识产权

### 性能优化
1. **数据库存储**: 建议使用SQLite或PostgreSQL
2. **分布式爬取**: 大规模数据建议使用分布式架构
3. **缓存机制**: 使用Redis缓存频繁访问的数据
4. **增量更新**: 实现增量爬取避免重复数据

## 故障排除

### 常见问题

1. **ChromeDriver版本不匹配**
   ```
   错误: This version of ChromeDriver only supports Chrome version XX
   解决: 更新ChromeDriver或降级Chrome浏览器
   ```

2. **Selenium连接失败**
   ```
   错误: Unable to establish websocket connection
   解决: 检查网络连接，尝试使用代理
   ```

3. **页面加载超时**
   ```
   错误: TimeoutException
   解决: 增加等待时间，检查网站可访问性
   ```

4. **元素查找失败**
   ```
   错误: NoSuchElementException
   解决: 更新选择器，检查页面结构变化
   ```

## 示例应用

### 价格监控系统
```python
import schedule
import time
from crawlers.ecommerce.ecommerce_crawler import EcommerceCrawler

def monitor_prices():
    """定时监控价格"""
    crawler = EcommerceCrawler()
    
    # 监控目标商品
    products = crawler.run(
        platforms=["taobao", "jd"],
        category="electronics",
        max_pages=2,
        price_range={"min": 1000, "max": 5000}
    )
    
    # 分析价格变化

    for product in products:
        print(f"[{product['platform']}] {product['title']} - ¥{product['price']:.2f}")
    
    # 保存数据到数据库（可选）
    # save_to_database(products)
    
    return products



# 设置定时任务

schedule.every(6).hours.do(monitor_prices)

# 立即运行一次
monitor_prices()

# 启动调度器
while True:
    schedule.run_pending()
    time.sleep(60)
```
